GuardDiff blocks risky AI-generated diffs before they merge.

Local-first scanning for staged changes, pull requests, patch files, and workspaces, with built-in rules for leaked secrets, auth changes, unsafe config, dangerous package scripts, and MCP exposure.

Start

npm install -g guarddiff
guarddiff staged --fail-on high

Before Commit

Run guarddiff staged directly or install the generated pre-commit hook with guarddiff init --pre-commit.

Patch Review

Use guarddiff diff --stdin or guarddiff diff --file change.patch for CI glue and saved patch files.

Workspace Scan

Use guarddiff scan . to inspect local files, including config files that are risky even outside a diff.

GitHub Action

- uses: TKY-27/GuardDiff@v0.1.0
  with:
    fail-on: high
    post-comment: true
    sarif: true
            annotations: true
    rules-update-check: true
    allow-inline-suppressions: false

The Action writes SARIF for Code Scanning, upserts one PR comment, emits workflow annotations with remediation details, and reports available rule updates when the registry manifest has newer detector versions. External rule packs stay disabled by default in CI unless explicitly enabled for trusted branches. Pull request scans read config from the trusted base ref, and inline suppressions are disabled by default.

VS Code

The stable VS Code extension shells out to the GuardDiff CLI, maps active findings to diagnostics, and exposes a status-bar scan entry. Set guarddiff.scanOnSave to run a workspace scan after saves.

Built-In Rule Coverage

Category Rule Families
Secrets OpenAI, Anthropic, AWS, GitHub, Stripe, Supabase, Firebase config, private keys, high-entropy strings.
Diffs Auth removal and bypass, CORS wildcard, debug endpoints, dangerous shell execution, eval, sensitive logging.
Config Committed env files, plaintext env secrets, dangerous package lifecycle scripts, Firebase and Firestore rules open to everyone.
MCP Full home access, root access, auto-exec without approval, and unrestricted network access.

Rule Updates

Publish /rules/manifest.json with this static site so CI can compare installed detector versions with the latest registry metadata. The current manifest is available at rules/manifest.json.
